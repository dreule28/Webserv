About Webserv

A fully compliant HTTP/1.1 server written in C++, handling concurrent connections with non-blocking I/O.

Architecture

Webserv uses poll() for multiplexed, non-blocking I/O. Each connection is managed through a state machine that handles request parsing, routing, response generation, and CGI execution.

Configuration follows an Nginx-like syntax with server blocks, location matching, and per-route method restrictions.

Static Serving

Serves files with correct MIME types. Supports directory indexing, custom index files, and comprehensive media type detection.

CGI Support

Executes scripts via fork/pipe with non-blocking I/O. Supports Python, Perl, and shell interpreters with proper environment variable passing.

File Uploads

Handles POST requests with both Content-Length and chunked Transfer-Encoding. Configurable upload directories per location.

Error Handling

Custom error pages per server block. Proper HTTP status codes for all error conditions including 400, 403, 404, 405, and 500.
